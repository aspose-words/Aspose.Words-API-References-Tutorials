Вы когда-нибудь оказывались по колено в море ревизий документов, задаваясь вопросом, кто что переместил и когда? Вы не одиноки. Обработка ревизий документов может быть утомительной задачей, особенно при работе с объемными документами. Но не волнуйтесь! С Aspose.Words для .NET вы можете легко идентифицировать и управлять этими ревизиями. В этом руководстве мы проведем вас через пошаговый процесс того, как получить типы ревизий слов в документе Word с помощью Aspose.Words для .NET. Итак, пристегните ремни и давайте нырнем!

## Предпосылки

Прежде чем приступить к написанию кода, вам понадобится несколько вещей:

1.  Библиотека Aspose.Words for .NET: если вы еще этого не сделали, загрузите ее с сайта[здесь](https://releases.aspose.com/words/net/).
2. Среда разработки: Visual Studio или любая другая совместимая с .NET IDE.

4.  Документ Word с изменениями: убедитесь, что у вас есть`.docx`файл с отслеживаемыми изменениями для тестирования кода.

## Импорт пространств имен

Для начала вам нужно импортировать необходимые пространства имен в ваш проект C#. Это позволит вам получить доступ к функциональным возможностям, предоставляемым Aspose.Words для .NET.

```csharp
using Aspose.Words;
using Aspose.Words.Revision;
using System;
```

Давайте разобьем пример на несколько шагов для лучшего понимания и реализации.

## Шаг 1: Настройте каталог документов

Прежде всего, вам нужно определить путь к каталогу ваших документов. Это то место, где будет находиться ваш документ Word с изменениями.

```csharp
string dataDir = "YOUR DOCUMENT DIRECTORY";
```

 Заменять`"YOUR DOCUMENT DIRECTORY"` с фактическим путем к папке с вашими документами.

## Шаг 2: Загрузите документ Word

Далее вам нужно загрузить документ Word в ваш проект. Этот документ должен иметь ревизии, которые вы хотите проанализировать.

```csharp
Document doc = new Document(dataDir + "Revisions.docx");
```

 Убедитесь, что файл`Revisions.docx` существует в указанном каталоге.

## Шаг 3: Получите доступ к коллекции абзацев

Теперь, когда ваш документ загружен, вам нужно получить доступ к абзацам в первом разделе тела документа. Это поможет вам пройтись по каждому абзацу, чтобы проверить наличие правок.

```csharp
ParagraphCollection paragraphs = doc.FirstSection.Body.Paragraphs;
```

## Шаг 4: Просмотрите абзацы и проверьте наличие изменений

Вот тут-то и происходит магия. Вы проходите по каждому абзацу и проверяете, был ли он перемещен (удален или вставлен).

```csharp
for (int i = 0; i < paragraphs.Count; i++)
{
    if (paragraphs[i].IsMoveFromRevision)
        Console.WriteLine("Paragraph {0} has been moved (deleted).", i);
    if (paragraphs[i].IsMoveToRevision)
        Console.WriteLine("Paragraph {0} has been moved (inserted).", i);
}
```

 Этот цикл проходит по каждому абзацу и использует`IsMoveFromRevision` и`IsMoveToRevision` свойства, позволяющие определить, был ли абзац перемещен (удален) или перемещен (вставлен).
