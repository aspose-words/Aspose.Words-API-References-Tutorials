في هذا المثال، سنشرح كيفية استخدام ميزة التعليمات البرمجية ذات المسافة البادئة مع Aspose.Words for .NET. يتم استخدام التعليمات البرمجية ذات المسافة البادئة لتمثيل كتل التعليمات البرمجية بشكل مرئي بتنسيق محدد.

## الخطوة 1: استخدام منشئ المستندات

أولاً، سنستخدم منشئ المستندات لإضافة محتوى إلى مستندنا.

```csharp
DocumentBuilder builder = new DocumentBuilder();
```

## الخطوة 2: إضافة نمط للتعليمات البرمجية ذات المسافة البادئة

 سنضيف نمطًا مخصصًا للرمز ذي المسافة البادئة باستخدام`Styles.Add` طريقة`Document` هدف. في هذا المثال، نقوم بإنشاء نمط يسمى "IndentedCode" للتعليمات البرمجية ذات المسافة البادئة.

```csharp
Style indentedCode = builder.Document.Styles.Add(StyleType.Paragraph, "IndentedCode");
builder.ParagraphFormat.Style = indentedCode;
```

## الخطوة 3: إضافة رمز بمسافة بادئة

يمكننا الآن إضافة كتلة تعليمات برمجية ذات مسافة بادئة باستخدام النمط المخصص "IndentedCode".

```csharp
builder.Writeln("This is an indented code block");
```

### مثال على التعليمات البرمجية المصدر للتعليمات البرمجية ذات مسافة بادئة باستخدام Aspose.Words لـ .NET

```csharp
// استخدم منشئ المستندات لإضافة محتوى إلى المستند.
DocumentBuilder builder = new DocumentBuilder();

Style indentedCode = builder.Document.Styles.Add(StyleType.Paragraph, "IndentedCode");
builder.ParagraphFormat.Style = indentedCode;
builder.Writeln("This is an indented code");
```

تهنئة ! لقد تعلمت الآن كيفية استخدام ميزة التعليمات البرمجية ذات المسافة البادئة مع Aspose.Words لـ .NET.


### الأسئلة الشائعة

#### س: ما هو رمز المسافة البادئة في Markdown؟

ج: التعليمات البرمجية ذات المسافة البادئة في Markdown هي طريقة تنسيق تُستخدم لعرض التعليمات البرمجية في مستند Markdown. وهو يتألف من وضع مسافة بادئة لكل سطر من التعليمات البرمجية بمسافات أو علامات تبويب.

#### س: كيفية استخدام التعليمات البرمجية ذات المسافة البادئة في Markdown؟

ج: لاستخدام التعليمات البرمجية ذات مسافة بادئة في Markdown، قم بوضع مسافة بادئة لكل سطر من التعليمات البرمجية بمسافات أو علامات تبويب.

#### س: ما هي مزايا التعليمات البرمجية ذات المسافة البادئة في Markdown؟

ج: تعمل التعليمات البرمجية ذات المسافة البادئة في Markdown على تحسين إمكانية قراءة التعليمات البرمجية وتسهل على القراء فهمها.

#### س: ما الفرق بين التعليمات البرمجية ذات المسافة البادئة وكتل التعليمات البرمجية في Markdown؟

ج: يتم استخدام التعليمات البرمجية ذات المسافة البادئة لمقتطفات التعليمات البرمجية الصغيرة التي يتم إدراجها في النص، بينما يتم استخدام كتل التعليمات البرمجية لعرض أجزاء أكبر من التعليمات البرمجية بتنسيق منفصل.

#### س: هل التعليمات البرمجية ذات المسافة البادئة في Markdown مدعومة من قبل جميع محرري Markdown؟

ج: قد يختلف دعم التعليمات البرمجية ذات المسافة البادئة في Markdown بين محرري Markdown. تحقق من الوثائق المحددة للناشر للتأكد.
