## استيراد مساحات الأسماء

لاستخدام Aspose.Words لـ .NET، تحتاج إلى تضمين مساحات الأسماء الضرورية في مشروعك. إليك مقتطف سريع للبدء:

```csharp
using Aspose.Words;
using Aspose.Words.Vba;
```

تتضمن مساحات الأسماء هذه جميع الفئات والطرق التي سنستخدمها في هذا البرنامج التعليمي.

## الخطوة 1: إعداد دليل المستندات الخاص بك

أول الأشياء أولاً، نحتاج إلى إعداد المسار إلى دليل المستندات الخاص بك. هذا هو المكان الذي يتم فيه تخزين مستندات Word الخاصة بك والمكان الذي ستحفظ فيه الملفات المعدلة.

### تحديد المسار

لنبدأ بتحديد المسار:

```csharp
string dataDir = "YOUR DOCUMENT DIRECTORY";
```

 يستبدل`"YOUR DOCUMENT DIRECTORY"` مع المسار الفعلي إلى المستندات الخاصة بك. هذا هو المكان الذي سيتواجد فيه مستندك المصدر مع مشروع VBA والمكان الذي سيتم فيه حفظ المستند الجديد.

## الخطوة 2: تحميل المستند باستخدام مشروع VBA

الآن بعد أن قمنا بإعداد الدليل الخاص بنا، حان الوقت لتحميل مستند Word الذي يحتوي على مشروع VBA. تعتبر هذه الخطوة حاسمة لأنها تتيح لنا الوصول إلى وحدات VBA ومعالجتها داخل المستند.

### تحميل الوثيقة

إليك كيفية تحميل المستند الخاص بك:

```csharp
Document doc = new Document(dataDir + "VBA project.docm");
```

يقوم مقتطف التعليمات البرمجية هذا بتحميل مستند Word المسمى "VBA project.docm" من الدليل المحدد.

## الخطوة 3: إنشاء مستند جديد

بعد تحميل المستند الأصلي، فإن الخطوة التالية هي إنشاء مستند جديد حيث سنقوم باستنساخ وحدة VBA. ستكون هذه الوثيقة الجديدة بمثابة الوجهة لمشروع VBA الخاص بنا.

### تهيئة المستند الجديد

إليك الكود لإنشاء مستند جديد:

```csharp
Document destDoc = new Document { VbaProject = new VbaProject() };
```

 يؤدي هذا إلى إنشاء مثيل جديد لـ`Document` فئة مع مشروع VBA فارغ.

## الخطوة 4: استنساخ وحدة VBA

الآن يأتي الجزء المثير، وهو استنساخ وحدة VBA من المستند الأصلي. تتضمن هذه الخطوة نسخ وحدة نمطية معينة وإضافتها إلى مشروع VBA الخاص بالمستند الجديد.

### الاستنساخ وإضافة الوحدة

دعونا نحلل الكود:

```csharp
VbaModule copyModule = doc.VbaProject.Modules["Module1"].Clone();
destDoc.VbaProject.Modules.Add(copyModule);
```

في السطر الأول، قمنا باستنساخ الوحدة المسماة "Module1" من مشروع VBA للمستند الأصلي. في السطر الثاني، نضيف هذه الوحدة المستنسخة إلى مشروع VBA للمستند الجديد.

## الخطوة 5: حفظ المستند الجديد

لقد قمنا بكل العمل الشاق، والآن حان الوقت لحفظ المستند الجديد باستخدام وحدة VBA المستنسخة. هذه الخطوة واضحة ولكنها ضرورية للحفاظ على تغييراتك.

### حفظ الوثيقة

إليك الرمز لحفظ المستند الخاص بك:

```csharp
destDoc.Save(dataDir + "WorkingWithVba.CloneVbaModule.docm");
```

يحفظ هذا السطر المستند الجديد بالاسم "WorkingWithVba.CloneVbaModule.docm" في الدليل المحدد.
