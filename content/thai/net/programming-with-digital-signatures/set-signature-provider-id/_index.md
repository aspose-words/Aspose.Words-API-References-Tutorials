ในบทช่วยสอนนี้ เราจะแนะนำคุณตลอดขั้นตอนในการใช้ฟีเจอร์ตั้งค่า ID ผู้ให้บริการลายเซ็นกับ Aspose.Words สำหรับ .NET คุณลักษณะนี้ช่วยให้คุณสามารถระบุ ID ผู้ให้บริการลายเซ็นสำหรับบรรทัดลายเซ็นในเอกสาร Word ทำตามขั้นตอนด้านล่าง:

## ขั้นตอนที่ 1: กำลังโหลดเอกสารและเข้าถึงบรรทัดลายเซ็น

เริ่มต้นด้วยการอัปโหลดเอกสารที่มีบรรทัดลายเซ็น:

```csharp
string dataDir = "YOUR DOCUMENT DIRECTORY";
Document doc = new Document(dataDir + "Signature line.docx");

SignatureLine signatureLine = ((Shape)doc.FirstSection.Body.GetChild(NodeType.Shape, 0, true)).SignatureLine;
```

## ขั้นตอนที่ 2: การตั้งค่าตัวเลือกลายเซ็น

สร้างอินสแตนซ์ของคลาส SignOptions และตั้งค่าตัวเลือกการลงนาม รวมถึง ID ผู้ให้บริการ:

```csharp
SignOptions signOptions = new SignOptions
{
ProviderId = signatureLine.ProviderId,
 SignatureLineId = signatureLine.Id
};
```

## ขั้นตอนที่ 3: การลงนามในเอกสาร

หากต้องการลงนามในเอกสาร คุณต้องใช้คลาส DigitalSignatureUtil และระบุใบรับรองการลงนาม:

```csharp
CertificateHolder certHolder = CertificateHolder.Create(dataDir + "morzal.pfx", "aw");

DigitalSignatureUtil.Sign(dataDir + "Digitally signed.docx",
	dataDir + "SignDocuments.SetSignatureProviderId.docx", certHolder, signOptions);
```

ตรวจสอบให้แน่ใจว่าได้ระบุเส้นทางที่ถูกต้องสำหรับเอกสาร ใบรับรอง และเอกสารที่ลงนาม

### ตัวอย่างซอร์สโค้ดสำหรับตั้งค่ารหัสผู้ให้บริการลายเซ็นโดยใช้ Aspose.Words สำหรับ .NET

นี่คือซอร์สโค้ดที่สมบูรณ์เพื่อตั้งค่า ID ผู้ให้บริการลายเซ็นด้วย Aspose.Words สำหรับ .NET:

```csharp

	// เส้นทางไปยังไดเร็กทอรีเอกสาร
	string dataDir = "YOUR DOCUMENT DIRECTORY";
	Document doc = new Document(dataDir + "Signature line.docx");

	SignatureLine signatureLine =
		((Shape) doc.FirstSection.Body.GetChild(NodeType.Shape, 0, true)).SignatureLine;

	SignOptions signOptions = new SignOptions
	{
		ProviderId = signatureLine.ProviderId, SignatureLineId = signatureLine.Id
	};

	CertificateHolder certHolder = CertificateHolder.Create(dataDir + "morzal.pfx", "aw");

	DigitalSignatureUtil.Sign(dataDir + "Digitally signed.docx",
		dataDir + "SignDocuments.SetSignatureProviderId.docx", certHolder, signOptions);

```

กรอก Signature Provider ID ในเอกสาร Word ของคุณด้วย Aspose.Words for .NET


## บทสรุป

### คำถามที่พบบ่อยสำหรับการตั้งค่า ID ผู้ให้บริการลายเซ็นในเอกสาร Word

#### ถาม: ID ผู้ให้บริการลายเซ็นในเอกสาร Word คืออะไร

ตอบ: ID ผู้ให้บริการลายเซ็นในเอกสาร Word คือตัวระบุเฉพาะที่ระบุผู้ให้บริการลายเซ็นดิจิทัล ช่วยระบุเอนทิตีหรือองค์กรที่รับผิดชอบในการสร้างและจัดการลายเซ็นดิจิทัล

#### ถาม: ฉันจะตั้งค่า ID ผู้ให้บริการลายเซ็นสำหรับบรรทัดลายเซ็นในเอกสาร Word โดยใช้ Aspose.Words สำหรับ .NET ได้อย่างไร

ตอบ: เมื่อต้องการตั้งค่า ID ผู้ให้บริการลายเซ็นสำหรับบรรทัดลายเซ็นในเอกสาร Word โดยใช้ Aspose.Words สำหรับ .NET คุณสามารถทำตามขั้นตอนเหล่านี้:
1.  โหลดเอกสารโดยใช้`Document` และระบุเส้นทางไปยังไฟล์เอกสาร
2.  เข้าถึงบรรทัดลายเซ็นโดยใช้วิธีการหรือคุณสมบัติที่เหมาะสม ตัวอย่างเช่นคุณสามารถใช้`GetChild` วิธีการดึงรูปร่างเส้นลายเซ็น
3. ดึง ID ผู้ให้บริการจากบรรทัดลายเซ็น
4.  สร้างอินสแตนซ์ของ`SignOptions` คลาสและตั้งค่า`ProviderId` คุณสมบัติเป็น ID ผู้ให้บริการที่ดึงข้อมูล
5.  ใช้`DigitalSignatureUtil.Sign` วิธีการลงนามในเอกสาร โดยระบุพารามิเตอร์ที่จำเป็นรวมถึง`SignOptions` วัตถุ วัตถุ

#### ถาม: ฉันจะเข้าถึงบรรทัดลายเซ็นในเอกสาร Word โดยใช้ Aspose.Words สำหรับ .NET ได้อย่างไร

 ตอบ: ในการเข้าถึงบรรทัดลายเซ็นในเอกสาร Word โดยใช้ Aspose.Words สำหรับ .NET คุณสามารถใช้วิธีการหรือคุณสมบัติที่เหมาะสมเพื่อดึงรูปร่างเส้นลายเซ็นจากโครงสร้างของเอกสารได้ ตัวอย่างเช่น คุณสามารถใช้`GetChild` วิธีการที่มีพารามิเตอร์ที่เหมาะสมเพื่อให้ได้รูปร่างเส้นลายเซ็นที่ต้องการ

#### ถาม: ฉันสามารถตั้งค่า ID ผู้ให้บริการลายเซ็นสำหรับบรรทัดลายเซ็นหลายบรรทัดในเอกสาร Word ได้หรือไม่

 ตอบ: ได้ คุณสามารถตั้งค่า ID ผู้ให้บริการลายเซ็นสำหรับบรรทัดลายเซ็นหลายบรรทัดในเอกสาร Word ได้ คุณสามารถวนซ้ำชุดบรรทัดลายเซ็นในเอกสารและตั้งค่า ID ผู้ให้บริการสำหรับบรรทัดลายเซ็นแต่ละบรรทัดทีละรายการโดยใช้`SignOptions.ProviderId` คุณสมบัติ.

#### ถาม: ID ผู้ให้บริการลายเซ็นในเอกสาร Word มีจุดประสงค์อะไร

ตอบ: ID ผู้ให้บริการลายเซ็นในเอกสาร Word มีจุดประสงค์เพื่อระบุเอนทิตีหรือองค์กรที่รับผิดชอบในการสร้างและจัดการลายเซ็นดิจิทัล จะช่วยสร้างความถูกต้องและความน่าเชื่อถือของลายเซ็นดิจิทัลโดยเชื่อมโยงกับผู้ให้บริการเฉพาะราย

#### ถาม: ใบรับรองดิจิทัลประเภทใดที่สามารถใช้เพื่อตั้งค่า ID ผู้ให้บริการลายเซ็นในเอกสาร Word

ตอบ: คุณสามารถใช้ใบรับรองดิจิทัล X.509 พร้อมข้อมูลผู้ให้บริการที่เหมาะสมเพื่อตั้งค่า ID ผู้ให้บริการลายเซ็นในเอกสาร Word ใบรับรองดิจิทัลควรออกโดยผู้ออกใบรับรองที่เชื่อถือได้ (CA) และมีข้อมูลเมตาที่จำเป็นในการระบุผู้ให้บริการ
