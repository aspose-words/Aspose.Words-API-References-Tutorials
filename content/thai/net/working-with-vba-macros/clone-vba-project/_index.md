## นำเข้าเนมสเปซ

ในการเริ่มต้น คุณจะต้องนำเข้าเนมสเปซที่จำเป็นจาก Aspose.Words เนมสเปซเหล่านี้มีคลาสและวิธีการที่คุณจะใช้ตลอดบทช่วยสอนนี้

ต่อไปนี้คือวิธีการนำเข้า:

```csharp
using Aspose.Words;
using Aspose.Words.Vba;
```

บรรทัดเหล่านี้รวมฟังก์ชันทั้งหมดที่เราต้องการในการจัดการเอกสาร Word และโครงการ VBA

## ขั้นตอนที่ 1: การตั้งค่าไดเร็กทอรีเอกสารของคุณ

ก่อนอื่น เราต้องกำหนดเส้นทางไปยังไดเร็กทอรีเอกสารของคุณ นี่คือที่ที่เอกสาร Word ต้นฉบับของคุณและเอกสารใหม่จะถูกบันทึกไว้

### การกำหนดเส้นทาง

เริ่มต้นด้วยการตั้งค่าเส้นทางไปยังไดเร็กทอรีของคุณ:

```csharp
string dataDir = "YOUR DOCUMENT DIRECTORY";
```

 แทนที่`"YOUR DOCUMENT DIRECTORY"` ด้วยเส้นทางจริงที่เก็บเอกสาร Word ของคุณ ไดเรกทอรีนี้จะเป็นพื้นที่ทำงานของเราสำหรับบทช่วยสอนนี้

## ขั้นตอนที่ 2: กำลังโหลดเอกสาร Word

เมื่อตั้งค่าไดเรกทอรีแล้ว ก็ถึงเวลาโหลดเอกสาร Word ที่มีโครงการ VBA ที่คุณต้องการโคลน ขั้นตอนนี้มีความสำคัญอย่างยิ่งในการเข้าถึงโครงการ VBA ภายในเอกสาร

### กำลังโหลดเอกสาร

ต่อไปนี้คือวิธีการโหลดเอกสารของคุณ:

```csharp
Document doc = new Document(dataDir + "VBA project.docm");
```

รหัสนี้จะโหลดเอกสาร Word ชื่อ "VBA project.docm" จากไดเร็กทอรีที่คุณระบุลงใน`doc` วัตถุ.

## ขั้นตอนที่ 3: การโคลนโครงการ VBA

ตอนนี้เราได้โหลดเอกสารต้นฉบับแล้ว ขั้นตอนต่อไปคือการโคลนโครงการ VBA ทั้งหมด ซึ่งหมายถึงการคัดลอกโมดูล ข้อมูลอ้างอิง และการตั้งค่าทั้งหมดจากเอกสารต้นฉบับไปยังเอกสารใหม่

### การโคลนโครงการ VBA

ลองดูรหัส:

```csharp
Document destDoc = new Document { VbaProject = doc.VbaProject.Clone() };
```

 ในบรรทัดนี้ เรากำลังสร้างเอกสารใหม่`destDoc` และตั้งค่าโครงการ VBA ให้เป็นโคลนของโครงการ VBA`doc`- ขั้นตอนนี้จะทำซ้ำเนื้อหา VBA ทั้งหมดจากเอกสารต้นฉบับไปเป็นเนื้อหาใหม่

## ขั้นตอนที่ 4: บันทึกเอกสารใหม่

เมื่อโคลนโครงการ VBA สำเร็จแล้ว ขั้นตอนสุดท้ายคือการบันทึกเอกสารใหม่ ขั้นตอนนี้ช่วยให้แน่ใจว่าการเปลี่ยนแปลงทั้งหมดของคุณยังคงอยู่และเอกสารใหม่พร้อมใช้งาน

### กำลังบันทึกเอกสาร

นี่คือรหัสสำหรับบันทึกเอกสารใหม่ของคุณ:

```csharp
destDoc.Save(dataDir + "WorkingWithVba.CloneVbaProject.docm");
```

บรรทัดนี้จะบันทึกเอกสารใหม่ด้วยโปรเจ็กต์ VBA ที่โคลนเป็น "WorkingWithVba.CloneVbaProject.docm" ในไดเร็กทอรีที่คุณระบุ
